4 This repository contains a C++ template library that essentially counts sufficient statistics
5 on binary arrays. The idea of the library is that this can be used together to build exponential
6 family models as those in Exponential Random Graph Models (ERGMs), but as a generalization that
7 also deals with non square arrays.
8 
9 # Design considerations
10 
11 ## Data structures
12 
13 For start, the main class object should hold the following:
14 
15 * **The array structure** Right now, we are thinking on dealing with an `std::unordered_map` type
16  of structure since search/addition/removel operations have constant average time.
17 
18 * **Pointer to undefined structure** Besides of the graph itself, the data may be acompained
19  by other datasets, for example, in the case of genetic annotation we may have the current
20  state of some genes, i.e., a binary vector.
21 
22 * **Structural constrains** Having arrays specifying the blocked spaces of the array. This,
23  in principle, could affect all operations related to modifying the array, e.g. if the
24  pair `(i, j)` is blocked, then no addition/deletion can be done on that respect.
25 
26  The structural constrains may be better reflected as a counterpart: free blocks. This
27  way, any algorithm that needs to iterate through cells that can be changed can use its
28  counter part.
29 
30  Enumeration of both sets would have in total $n \times m$ unordered pairs. One problem of this
31  is the fact that this type of data structure is unefficient as it can grow too fast. Yet,
32  the whole idea of this C++ library is to be able to fully enumerate support of arrays, so
33  problems that need to deal with larger datasets may not be suitable for this approach.
34 
35 ## Algorithms to implement
36 
37 * **Counters** Users should be able to define counters using change statistics. From the
38  ERGM literature, we know that change statistics can be a very efficient way of counting
39  when we have a Markov process. In our case, since we will be doing exhaustive ennumeration,
40  a good an efficient way of counting statistics is counting as we add/remove zeros.
41 
42  Counters should have the following implementation:
43 
44  ```cpp
45  double counter_[name](const Array & x, unsigned int row, unsigned int col) {
46  // Visit neighbors
47  Array::local_iterator iter(x, row, col);
48  double counts = 0;
49  for (auto i = iter.begin(); i != iter.end(); ++i) {
50  ...[do your thing]...
51  counts += ...
52  }
53 
54  return counts;
55  }
56  ```
57 
58 * **Constrained Exhaust enumeration** Exhaust enumeration can be done using recursive algorithms
59  activating and deactivating cells in the array. One nice feature would be to allow users
60  to specify constrained, essentially blocked, cells in the array. This would go together with
61  the counters.
62 
63  The constrains can just be `std::unordered_map` objects in which the coordinates of the
64  cells that need to be blocked are specified. Some standard constrains can be:
65 
66  - Blocks (ranges).
67  - Symmetry (in the case of square, undirected graphs).
68 
69  Furthermore, we should, at least in principle, allow the user to speficy default values
70  for the blocked cells (0/1).
71 
72 * **Array changes** Addition and deletion of 0/1 states. If we use `std::unordered_map`
73  this should be straight forward. Perhaps just making an alias or binary operator, e.g.
74 
75  ```cpp
76  X += (i, j);
77  X.add(i, j);
78  X.rm(i, j);
79  ```
80 
81 * **Markov Chains** This would be nice to have, but not necesary for now. The idea
82  is to have various types of algorithms to implement transitions to new states,
83  for example
84 
85  - Randomly adding/removing new pairs.
86  - Unconstrained endpoints-switching/swap.
87  - Constrained endpoints-swap.
88 
89  The constrained component can be related to the graph constrains specified part
90  of the countner.
91 
92 * **Estimate support size** Do this conditioning on the constrains. This should be
93  rather straight forward. The support of the set should be defined by
94 
95  $$
96  2^{(n\times m - |blocked|)}
97  $$
98 
99  Where $blocked$ is the set of blocked cells.
